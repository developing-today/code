Use `chrome-devtools_*` MCP tools to verify changes and inspect website state — navigate pages, take snapshots/screenshots, inspect console messages and network requests. Prefer chrome-devtools for agent verification of UI changes and website state over assumptions. After validating a change via chrome-devtools, write Playwright e2e tests that recreate the same verification so it persists as automated coverage. If chrome-devtools tools fail, fall back to curl/webfetch for basic HTTP checks.
